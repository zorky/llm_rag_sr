OOS Recommendations and Limits

This page provides key recommendations aimed to help you maximize the performance and benefits of our object storage solution.

Recommendations

For an optimal use of your buckets, we recommend:

Usage Recommendation

Buckets

100 per account (could be extended to 1000)

Size

5TB for storable objects

Versions

10 million, including delete markers, in a single bucket

Downloads per second, for GET/HEAD requests

300 for a single bucket

Downloads per second, for PUT/COPY/POST/DELETE requests

200 for a single bucket

Hard Limits

For objects:

Constraint Description

Upload time

Complete uploads within 3 minutes to avoid timeouts.

Request timeouts are influenced by the time it takes to upload a file: the larger the file and the slower the upload speed, the higher the risk of exceeding the timeout limit. To upload large objects, we recommend you favor uploading your objects in part. For more information, see Using Multipart Upload.

Tags per object

Maximum of 10 tags per object.

Tag size

Maximum of 128 characters for key size and 256 characters for value size.

ACLs per bucket

Only one ACL per bucket. For more information, see Configuring an Object ACL.
