January 23, 2025 New Hourly and Yearly Pricing Mode per VM Type

  • Marketplace

January 15, 2024 TINA OS Update

Unique Identifiers Added to Keypairs

  • New feature

  • Compute

To enhance consistency across our cloud resources and ease their management, keypairs now include a unique identifier (ID) available as a KeypairId attribute in the OUTSCALE API.

The new KeypairId attribute is not a substitute of the current KeypairName attribute. The KeypairName attribute remains fully functional.

Each keypair created receives its ID upon creation.
For existing keypairs, they will be assigned an ID without any impact on your usage.

Tags Added to Keypairs

  • New feature

  • Compute

Keypairs now support the use of tags.

Tags are labels that enable you to categorize your resources and identify them. For example, you can use tags to make inventories of your resources, as you can filter resources that share the same tag. For more information, see About Tags and Adding or Removing Tags.

Behavior of CreateNetPeering (OUTSCALE API)

  • Improvement

  • Compute

In the previous TINA OS update, the AccepterOwnerId parameter was added to the CreateNetPeering OUTSCALE API method as optional.

From this update, in the case of sharing a Net peering connection between two Nets belonging to two different users, the AccepterOwnerId parameter is now mandatory.

Starting Virtual Machines (VMs)

  • Improvement

  • Compute

We are continuously improving the performances of the OUTSCALE Cloud, especially the duration VMs start times.

The first visible advances are available in this release of TINA OS.

Many more improvements are still on the way in the coming months.

Parameter Removed From the Flexible Compute Unit (FCU) API

  • Removal

  • Compute

In the FCU API, it is no longer possible to get the product types associated to snapshots.

Therefore, the SnapshotId parameter is removed from the GetProductType and GetProductTypes FCU API methods.

Matching Behavior Between CreateNetPeering (OUTSCALE API) and CreateVpcPeeringConnection (FCU API)

  • Improvement

  • API / tools

The PeerOwnerId parameter of the CreateVpcPeeringConnection FCU method is now added to its corresponding OUTSCALE API CreateNetPeering method under the name AccepterOwnerId.

This integration is done in two steps:

  1. In this version, the AccepterOwnerId parameter is added as optional in the OUTSCALE API. It is included in this release to enable users and tools to integrate it as of now.

  2. In a future version, this parameter will be mandatory in the case of sharing a Net peering connection between two Nets belonging to two different users.

Consistency of the IsSourceDestChecked Parameter

  • Improvement

  • API / tools

In the OUTSCALE API, the value of the IsSourceDestChecked parameter in the primary Network Interface Card (NIC), with 0 as device index, is now consistent with the IsSourceDestChecked parameter of the VM.

This improvement is also applied in the Flexible Compute Unit (FCU) API with the sourceDestCheck parameter of the Flexible Network Interface (FNI) that is consistent with the sourceDestCheck parameter of the instance.

June 17, 2024 TINA OS Update

CPU Performances

  • Improvement

  • Compute

CPU performance has been improved across all our Regions.

The placement of vCores of new virtual machines has been optimized, through better support for NUMA (non-uniform memory access) zones in Intel processors.

Network Performances

  • Improvement

  • Network

The network performance has been improved across all our Regions.

For the newly created Nets, the network can now handle a larger number of packets per second, and bandwidth capacity has been increased.

Fixes and Minor Improvements

  • Improvement

  • API / tools

  • In the RouteTable methods of the OUTSCALE API, the GatewayId parameter can now return the local value.

  • In the OUTSCALE API, the creation of a Net peering between a Net you own and a Net that belongs to another account has been fixed.

  • You can no longer use outscale-elb-sg as the name of a security group. This name is reserved to OUTSCALE’s usage.

  • The performance of the FCU API DescribeInstances method has been improved. You can now get information from a larger number of VMs simultaneously.
