Gérer les versions d’un produit OMI

En tant que vendeur partenaire sur OUTSCALE Marketplace, vous avez accès aux tableaux Versions de vos produits OMI qui listent une ou plusieurs versions, leurs ID, versions de logiciels, dates de modification et statuts de publication. À partir de ces tableaux, vous pouvez publier, modifier et dépublier les versions des produits OMI. Les souscripteurs et les administrateurs de leurs organisations sont automatiquement informés par e-mail lorsque des versions sont publiées et dépubliées.

Lorsque vous publiez une version d’un produit OMI sur la Marketplace, vous devez d’abord créer une OMI sur le Cloud OUTSCALE.

Lorsqu’une nouvelle version est publiée, une copie de l’OMI correspondante est automatiquement créée, avec un product code spécifique associé. Pour en savoir plus, voir Créer et publier un produit OMI. L’accès à cette copie d’OMI est automatiquement donné aux souscripteurs existants afin de leur permettre de créer des machines virtuelles à partir de celle-ci. Pour en savoir plus, voir À propos des OMI > Permissions, copies et exports d’OMI vers OOS.

À la publication d’une version, sa date de modification est mise à jour sur la page produit correspondante et dans le tableau Mes produits. 3DS OUTSCALE recommande par e-mail aux souscripteurs et aux administrateurs de leurs organisations de mettre à niveau vers les dernières versions publiées. Lorsqu’une version est modifiée, la date de modification est mise à jour dans le tableau Versions.

Vous pouvez dépublier une version car elle a atteint sa fin de vie ou pour un problème de sécurité. À la dépublication d’une version, l’OMI correspondante est automatiquement supprimée du Cloud OUTSCALE, mais les souscripteurs peuvent continuer à utiliser les machines virtuelles (VM) déjà créées depuis l’OMI. La date de modification et le statut de publication de la version sont mis à jour dans le tableau Versions du produit correspondant. La dernière version disponible est également mise à jour sur la page produit.

Vous pouvez également dépublier un produit OMI entièrement, c’est-à-dire toutes ses versions. Pour en savoir plus, voir Dépublier un produit.

Publier une version

  1. Dans le coin supérieur droit de l’interface, cliquez sur votre nom d’utilisateur ou votre adresse e-mail.
    Un menu déroulant apparaît.

  2. Cliquez sur Mes produits.
    La page Mes produits apparaît.

  3. Cliquez sur Modifier sur la ligne produit souhaitée.
    La page produit correspondante apparaît.

  4. Cliquez sur Accéder dans la section Versions.
    Le tableau Versions apparaît.

  5. Cliquez sur Créer une version.
    Le formulaire Versions est affiché et certains champs sont préremplis sur la base de la dernière version créée.

  6. Dans la section Informations de l’OMI :

    1. Sélectionnez le nom ou l’ID de l’OMI approprié.
      Le nom de l’OMI et l’ID de la copie de l’OMI sont automatiquement renseignés.

      Le nom ou l’ID de l’OMI ne peuvent pas être modifiés après la publication de la version, mais vous pouvez publier de nouvelles versions de votre produit OMI avec de nouveaux noms ou ID d’OMI.

      • Les OMI sélectionnables sont celles qui sont créées via le compte TINA correspondant et sur la Région correspondante.

      • Pour un fonctionnement optimal sur la Marketplace, ces OMI doivent rester privées. Pour en savoir plus, voir À propos des OMI.

    2. (optionnel) Modifiez la version du logiciel.

    3. (optionnel) Modifiez le système d’exploitation.

    4. (optionnel) Modifiez la base logicielle en ajoutant des éléments.

    5. Saisissez une note de release.

    6. (optionnel) Modifiez les instructions d’utilisation de votre produit.

  7. Cliquez sur Publier.
    La boîte de dialogue Copie de l’OMI apparaît.

  8. Cliquez sur Poursuivre la publication.
    La demande de publication est envoyée à 3DS OUTSCALE et nécessite un délai de traitement. La version est au statut et ne pourra pas être modifiée avant d’être au statut publiée.

Modifier une version

Vous ne pouvez pas modifier :

  • Une version d’un produit OMI dont le statut de publication est dépublié.

  • Le nom ou l’ID de l’OMI d’une version d’un produit publié.

  1. Dans le coin supérieur droit de l’interface, cliquez sur votre nom d’utilisateur ou votre adresse e-mail.
    Un menu déroulant apparaît.

  2. Cliquez sur Mes produits.
    La page Mes produits apparaît.

  3. Cliquez sur Modifier sur la ligne produit souhaitée.
    La page produit correspondante apparaît.

  4. Cliquez sur Accéder dans la section Versions.
    Le tableau Versions apparaît.

  5. Cliquez sur Modifier sur la ligne version souhaitée.
    Le formulaire Versions correspondant apparaît.

  6. Modifiez les informations souhaitées et cliquez sur Publier les modifications.
    Les modifications sont publiées sur la Marketplace.

Dépublier une version

Vous ne pouvez pas dépublier l’unique version restante d’un produit.

Dans le cas d’une fin de vie

  1. Dans le coin supérieur droit de l’interface, cliquez sur votre nom d’utilisateur ou votre adresse e-mail.
    Un menu déroulant apparaît.

  2. Cliquez sur Mes produits.
    La page Mes produits apparaît.

  3. Cliquez sur Modifier sur la ligne produit souhaitée.
    La page produit correspondante apparaît.

  4. Cliquez sur Accéder dans la section Versions.
    Le tableau Versions apparaît.

  5. Cliquez sur Dépublier sur la ligne de version souhaitée.
    Une popup apparaît.

  6. Dans la liste Motif de dépublication, sélectionnez Fin de vie de version et cliquez sur Dépublier.
    La version est dépubliée et les souscripteurs et les administrateurs de leurs organisations sont informés par e-mail.

Dans le cas d’un problème de sécurité

  1. Dans le coin supérieur droit de l’interface, cliquez sur votre nom d’utilisateur ou votre adresse e-mail.
    Un menu déroulant apparaît.

  2. Cliquez sur Mes produits.
    La page Mes produits apparaît.

  3. Cliquez sur Modifier sur la ligne produit souhaitée.
    La page produit correspondante apparaît.

  4. Cliquez sur Accéder dans la section Versions.
    Le tableau Versions apparaît.

  5. Cliquez sur Dépublier sur la ligne de version souhaitée.
    Une popup apparaît.

  6. Dans la liste Motif de dépublication, sélectionnez Problème de sécurité et cliquez sur Suivant.

  7. Spécifiez :

    1. Le problème de sécurité.

    2. (optionnel) Le correctif.

    3. La version de remplacement.

  8. Cliquez sur Dépublier.
    La version est dépubliée et les souscripteurs et les administrateurs de leurs organisations sont informés par e-mail.
