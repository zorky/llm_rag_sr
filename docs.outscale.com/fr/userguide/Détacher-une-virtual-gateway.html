Détacher une virtual gateway

Vous pouvez détacher une virtual gateway d’un Net.

Cette action modifie l’état d’attachement de la virtual gateway de attached en detached, mais l’état de la virtual gateway reste available. Pour en savoir plus, voir À propos des virtual gateways > Cycle de vie.

Détacher une virtual gateway avec Cockpit v2

  1. Dans le dashboard Virtual Gateways, cochez la case de la virtual gateway que vous voulez détacher d’un Net.
    La virtual gateway est sélectionnée.

  2. Cliquez sur  Détacher d’un Net.
    La boîte de dialogue DÉTACHER D’UN NET apparaît.

  3. Cliquez sur Détacher.
    La virtual gateway est détachée du Net.

Détacher une virtual gateway avec OSC CLI

À ce jour, cette section est disponible en anglais uniquement.

The UnlinkVirtualGateway command detaches a virtual gateway from a Net.
You must wait until the virtual gateway is in the detached state before you can attach another Net to it or delete the Net it was previously attached to.

Request sample
$ osc-cli api UnlinkVirtualGateway --profile "default" \
    --VirtualGatewayId "vgw-12345678" \
    --NetId "vpc-12345678"

This command contains the following attributes that you need to specify:

  • DryRun: (optional) If true, checks whether you have the required permissions to perform the action.

  • NetId: The ID of the Net from which you want to detach the virtual gateway.

  • VirtualGatewayId: The ID of the virtual gateway.

The UnlinkVirtualGateway command returns the following elements:

  • ResponseContext: Information about the context of the response.

    • RequestId: The ID of the request.

Result sample
{
  "ResponseContext": {
    "RequestId": "0475ca1e-d0c5-441d-712a-da55a4175157"
  }
}

Détacher une virtual private gateway avec AWS CLI

Avant de commencer : Installez et configurez AWS CLI. Pour en savoir plus, voir Installer et configurer AWS CLI.

Pour détacher une virtual private gateway d’un VPC, utilisez la commande detach-vpn-gateway en suivant cette syntaxe :

Exemple de requête
$ aws ec2 detach-vpn-gateway \
    --profile YOUR_PROFILE \
    --vpn-gateway-id vgw-00ac2c82 \
    --vpc-id vpc-eb1e7306 \
    --endpoint https://fcu.eu-west-2.outscale.com

Cette commande contient les attributs suivants que vous devez spécifier :

  • (optionnel) profile : Le profil nommé que vous voulez utiliser, créé pendant la configuration d’AWS CLI. Pour en savoir plus, voir Installer et configurer AWS CLI.

  • vpn-gateway-id : L’ID de la virtual private gateway que vous voulez détacher.

  • vpc-id : L’ID du VPC duquel vous voulez détacher la virtual private gateway.

  • endpoint : Le endpoint correspondant à la Région à laquelle vous voulez envoyer la requête. Pour en savoir plus, voir Installer et configurer AWS CLI.

La virtual private gateway spécifiée est détachée du VPC.
