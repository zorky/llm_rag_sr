15 janvier 2025 TINA OS Update

Identifiants uniques ajoutés aux keypairs

  • Nouveauté

  • Puissance de calcul

Pour améliorer la cohérence entre nos ressources cloud et faciliter leur gestion, les keypairs possèdent désormais un identifiant unique (ID) sous la forme d’un attribut KeypairId dans l’API OUTSCALE.

Ce nouvel attribut KeypairId ne remplace pas l’attribut KeypairName actuel. L’attribut KeypairName reste entièrement fonctionnel.

Chaque keypair créée a un ID attribué à sa création.
Pour les keypairs existantes, un ID leur sera attribué sans aucun impact sur votre utilisation.

Tags ajoutés aux keypairs

  • Nouveauté

  • Puissance de calcul

Les keypairs supportent désormais l’utilisation des tags.

Les tags sont des étiquettes qui vous permettent d’identifier et de classer vos ressources. Par exemple, vous pouvez utiliser les tags pour inventorier vos ressources, car ils vous permettent de filtrer celles qui partagent un même tag. Pour en savoir plus, voir À propos des tags et Ajouter ou retirer des tags.

Comportement de CreateNetPeering (OUTSCALE API)

  • Amélioration

  • Puissance de calcul

Lors de la précédente mise à jour de TINA OS, le paramètre AccepterOwnerId a été ajouté à la méthode API OUTSCALE CreateNetPeering comme optionnel.

A partir de cette mise à jour, dans le cas du partage d’un Net peering entre deux Nets appartenant à deux utilisateurs différents, le paramètre AccepterOwnerId est désormais obligatoire.

Démarrage des machines virtuelles (VMs)

  • Amélioration

  • Puissance de calcul

Nous améliorons continuellement les performances du Cloud OUTSCALE, en particulier la durée de démarrage des VMs.

Les premières améliorations visibles sont disponibles dans cette version de TINA OS.

De nombreuses autres améliorations sont encore à venir dans les prochains mois.

Paramètre retiré de l’API Flexible Compute Unit (FCU)

  • Suppression

  • Puissance de calcul

Dans l’API FCU, il n’est plus possible d’obtenir les types de produit associés aux snapshots.

Par conséquent, la paramètre SnapshotId est retiré des méthodes API FCU GetProductType et GetProductTypes.

16 septembre 2024 Mise à jour de TINA OS

Appliance dans un Net

  • Nouveauté

  • Puissance de calcul

Pour des raisons de sécurité, ou pour des raisons de capacité de vos solutions logicielles, vous souhaitez déléguer le contrôle des flux réseau à votre appliance.

Vous pouvez suivre ce tutoriel pour mettre en place une appliance dans un Net : Mettre en place une appliance dans un Net.

Harmonisation des comportements entre CreateNetPeering (API OUTSCALE) et CreateVpcPeeringConnection (API FCU)

  • Amélioration

  • API / outils

Le paramètre PeerOwnerId de la méthode API FCU CreateVpcPeeringConnection est désormais ajouté à la méthode API OUTSCALE correspondante CreateNetPeering sous le nom AccepterOwnerId.

Cette intégration est faite en deux étapes :

  1. Dans cette version, le paramètre AccepterOwnerId est ajouté comme optionnel dans l’API OUTSCALE. Il est inclus dans cette release pour permettre aux utilisateurs et aux outils de l’intégrer dès maintenant.

  2. Dans une prochaine version, ce paramètre sera obligatoire dans le cas d’un partage de connexion Net peering entre deux Nets appartenant à deux utilisateurs différents.

Harmonisation du paramètre IsSourceDestChecked

  • Amélioration

  • API / outils

Dans l’API OUTSCALE, la valeur du paramètre IsSourceDestChecked dans la Network Interface Card (NIC) principale, avec 0 comme index de périphérique, est désormais harmonisée avec le paramètre IsSourceDestChecked de la VM.

Cette amélioration est aussi appliquée dans l’API Flexible Compute Unit (FCU) avec le paramètre sourceDestCheck de la Flexible Network Interface (FNI) qui est harmonisé avec le paramètre sourceDestCheck de l’instance.

17 juin 2024 Mise à jour de TINA OS

Performances CPU

  • Amélioration

  • Puissance de calcul

Les performances des CPU ont été améliorées sur l’ensemble de nos Régions.

Le placement des vCores des nouvelles machines virtuelles est optimisé grâce à une meilleure prise en compte des zones NUMA (Non uniform memory access) dans les processeurs Intel.

Performances réseau

  • Amélioration

  • Réseau

Les performances réseau des Nets ont été améliorées sur l’ensemble de nos Régions.

Pour les Nets nouvellement créés, le réseau est désormais capable de traiter un plus grand nombre de paquets par seconde et la capacité de la bande passante a été augmentée.

Correctifs et améliorations mineures

  • Amélioration

  • API / outils

  • Dans les méthodes RouteTable de l’API OUTSCALE, le paramètre GatewayId peut désormais renvoyer la valeur local.

  • Dans l’API OUTSCALE, la création d’un Net peering entre un Net vous appartenant et celui d’un autre utilisateur a été corrigée.

  • Il n’est plus possible d’utiliser le nom outscale-elb-sg pour un security group. Ce nom est réservé l’usage d’OUTSCALE.

  • La performance de la méthode API FCU DescribeInstances a été améliorée et permet d’obtenir simultanément les informations d’un plus grand nombre de VM.
