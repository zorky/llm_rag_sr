À propos des options DHCP

Le Dynamic Host Configuration Protocol (DHCP, protocole de configuration dynamique des hôtes) est un protocole réseau qui attribue des paramètres de configuration réseau, tels que les IP, à travers des serveurs DHCP. Les Nets vous permettent de paramétrer des options DHCP pour les machines virtuelles (VM) que vous créez à l’intérieur.

Nets et options DHCP

Les VM que vous créez dans un Net reçoivent un nom de DNS privé par IP, et, si vous leur attachez une IP publique, un nom de DNS public. Ces noms de DNS sont configurés par un serveur de nom de domaine spécifié dans le groupe d’options DHCP associé au Net.

Lorsque vous créez un Net, un groupe d’options DHCP par défaut est automatiquement créé et associé au Net. Vous pouvez également créer votre propre groupe d’options personnalisé et l’associer au Net, ce qui remplace celui par défaut. Pour en savoir plus sur les options DHCP par défaut et les valeurs par défaut appliquées au Net, voir la section Groupe d’options DHCP par défaut ci-dessous.

Lorsque vous créez un groupe d’options DHCP, vous pouvez spécifier les informations suivantes :

  • Le serveur de nom de domaine (DNS, domain name server) et le nom de domaine à utiliser pour les VM créées dans le Net.
    Dans l’option domain-name-servers, vous pouvez spécifier les IP de un ou plusieurs serveurs de nom de domaine.
    Si vous spécifiez un nom de domaine personnalisé, vous pouvez également spécifier un nom d’hôte DNS personnalisé pour vos VM avec l’option domain-name.

    Pour utiliser le nom de domaine OUTSCALE dans un groupe d’options DHCP personnalisé, vous devez spécifier les valeurs suivantes :

    • domain-name-servers : OutscaleProvidedDNS

    • domain-name : <nom_région>.compute.internal

    Pour en savoir plus, voir la section Groupe d’options DHCP par défaut ci-dessous.

  • Le serveur NTP à utiliser pour synchroniser ces VM avec des machines dans d’autres réseaux, par exemple via un Net peering, une connexion VPN ou une connexion DirectLink.
    Dans l’option ntp-servers, vous pouvez spécifier les IP de un ou plusieurs serveurs NTP.

    • Si les serveurs NTP que vous spécifiez sont situés dans un réseau qui n’apparaît pas encore dans les route tables de votre Net, vous devez ajouter une route vers leurs IP. Pour en savoir plus, voir Créer une route.

    • Si vous ne spécifiez aucun serveur NTP et que vous créez une VM avec une OUTSCALE machine image (OMI) officielle, un serveur NTP OUTSCALE par défaut est utilisé. Ce serveur NTP est configuré dans l’OMI officielle et vous n’avez pas besoin d’ajouter de route vers celui-ci.

  • Le serveur de logs à utiliser pour publier les logs. Dans l’option log-servers, vous pouvez spécifier les IP de un ou plusieurs serveurs de logs.

    • Si les serveurs de logs que vous spécifiez sont situés dans un réseau qui n’apparaît pas encore dans les route tables de votre Net, vous devez ajouter une route vers leurs IP. Pour en savoir plus, voir Créer une route.

Les groupes d’options DHCP que vous créez sont disponibles pour votre compte et vous pouvez les associer à un ou plusieurs de vos Nets. Cependant, un Net ne peut avoir qu’un seul groupe d’options DHCP associé à la fois.

Vous ne pouvez pas modifier les options que contient un groupe d’options DHCP après sa création. Cependant, vous pouvez remplacer le groupe d’options DHCP associé à un Net par un autre, par défaut ou personnalisé. Pour en savoir plus, voir Modifier le groupe d’options DHCP associé à un Net.

Lorsque vous modifiez le groupe d’options DHCP associé à un Net, les options qu’il contient sont automatiquement appliquées aux VMs existantes, ainsi qu’aux VMs créées ultérieurement. Les VMs existantes appliquent ces options quelques heures après la modification, selon la fréquence de renouvellement de leur bail DHCP. Vous n’avez pas besoin de redémarrer ces VMs. Vous pouvez cependant forcer le renouvellement du bail DHCP avec la commande dhclient pour les VMs Linux, ou la commande ipconfig/renew pour les VMs Windows.

Vous pouvez supprimer à tout moment un groupe d’options DHCP que vous avez créé, mais vous ne pouvez pas supprimer celui par défaut pour votre Net. Pour en savoir plus, voir Supprimer un groupe d’options DHCP.

Groupe d’options DHCP par défaut

Le groupe d’options DHCP par défaut contient les options suivantes :

  • domain-name-servers : OutscaleProvidedDNS, chemin vers le nom de domaine OUTSCALE.

  • domain-name : le nom de domaine pour votre Région, au format <nom_région>.compute.internal. Par exemple, une VM avec l’IP privée 10.0.1.5 dans la Région eu-west-2 a pour nom d’hôte ip-10-0-1-5.eu-west-2.compute.internal.

Le groupe d’options DHCP par défaut ne contient aucun serveur NTP, mais un serveur NTP est automatiquement configuré dans les OMI officielles. Pour utiliser un autre serveur NTP, vous devez créer un groupe d’options DHCP contenant l’option ntp-servers, et ajouter la route correspondante vers le serveur si nécessaire. Pour en savoir plus, voir la section Nets et options DHCP ci-dessus.

Le serveur de nom de domaine OutscaleProvidedDNS appartient à 3DS OUTSCALE. La première IP du bloc CIDR de votre Net est réservée à ce serveur. Par exemple, dans un Net avec le bloc CIDR 10.0.0.0/16, l’IP du serveur est 10.0.0.1.
