Référence des tags user data

Cette page liste les tags OUTSCALE que vous pouvez utiliser pour gérer le comportement de vos ressources Cloud selon vos besoins matériel ou réseau.

Vous pouvez appliquer ces tags dans les metadata de vos machines virtuelles (VM), ou avec la méthode classique. Pour en savoir plus, voir Configurer une VM avec les user data et les tags OUTSCALE ou Ajouter ou retirer des tags.

Des tags sont disponibles pour les besoins suivants :

Placement des VM

Nom du tag Description Valeur du tag Applicabilité

osc.fcu.repulse_server

Place les VMs avec la même valeur de tag sur des serveurs différents

Libre

  • À la création de la VM

  • Après un arrêt/démarrage de la VM

osc.fcu.attract_server

Place les VMs avec la même valeur de tag sur le même serveur

osc.fcu.repulse_cluster

Place les VMs avec la même valeur de tag sur des clusters Cisco UCS différents

osc.fcu.attract_cluster

Place les VMs avec la même valeur de tag sur le même cluster Cisco UCS

Ces tags peuvent être utilisés en mode strict, ce qui signifie que si les options de placement demandées sont impossibles, une erreur Insufficientcapacity est renvoyée. Si vous n’utilisez pas le mode strict, vos demandes d’options de placement sont contournées en cas d’impossibilité.

Pour spécifier le mode strict, ajoutez _strict au nom du tag (par exemple, osc.fcu.repulse_server_strict).

Association des IP aux VM

Nom du tag Description Valeur du tag Applicabilité

osc.fcu.eip.auto-attach

Associe automatiquement une IP publique à une VM, qui la conserve ainsi à travers le processus d’arrêt et démarrage

L’IP publique que vous voulez associer.

Pour en savoir plus, voir À propos des IP publiques > Association des IP publiques.

  • À la création de la VM

  • Après un arrêt/démarrage de la VM

private_only

Empêche l’attribution d’une IP publique à une VM créée dans le Cloud public

true ou false

Contournement de pare-feu

Le contournement de pare-feu consiste à désactiver des security groups pour réduire la latence générale entre des VM. Pour en savoir plus, voir À propos des Nets > Routage et sécurité d’un Subnet.

Cas d’usage Nom du tag à appliquer sur le Net Nom du tag à appliquer sur les VM Effet

VM dans un Net

osc.fcu.disable_lan_security_groups

n/a

Les security groups sont désactivés entre les VM des même Subnets.

C’est le comportement par défaut lorsqu’aucun tag n’est appliqué.

osc.fcu.enable_lan_security_groups

n/a

Les security groups sont activés entre les VM des mêmes Subnets.

VM dans le Cloud public

/

osc.fcu.enable_lan_security_groups

Les security groups sont activés entre les VM.

C’est le comportement par défaut lorsqu’aucun tag n’est appliqué.

osc.fcu.disable_lan_security_groups

Les security groups sont désactivés entre les VM ayant le même tag et même valeur de tag.

  • Sur un Net, ces tags doivent être appliqués après la création du Net mais avant la création de Subnets. Une fois que vous avez créé un Subnet dans le Net, ces tags ne peuvent plus modifier le comportement.

  • Sur des VM, ces tags doivent être appliqués lors de la création des VM ou être suivis d’un arrêt/redémarrage.

  • Les valeurs de ces tags peuvent être n’importe quelle valeur, sauf pour osc.fcu.disable_lan_security_groups dans le Cloud public, où la valeur doit avoir une longueur d’au moins un caractère.

Utilisation simultanée de tags OUTSCALE et d’une section libre

Nom du tag Description Valeur du tag Applicabilité

filter_private_section

Filtre les tags OUTSCALE de la VM afin que vous puissiez utiliser simultanément une section de tags OUTSCALE et une section libre d’user data, dans la même VM.

Si true, tous vos tags OUTSCALE prennent effet, mais ne sont pas visibles lorsque vous décrivez les user data de la VM.

Si false ou non défini, vous ne pouvez pas utiliser simultanément une section de tags OUTSCALE et une section libre d’user data.

true ou false

  • À la création de la VM

  • Après un arrêt/démarrage de la VM

Ce tag est disponible uniquement pour l’API OUTSCALE. Il n’est pas compatible avec l’API FCU.
