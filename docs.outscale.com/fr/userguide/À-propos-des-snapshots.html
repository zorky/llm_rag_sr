À propos des snapshots

Un snapshot (instantané) est une image d’un volume à un moment donné qui vous permet de sauvegarder vos données, créer des répliques d’un volume au moment où le snapshot a été créé, ou déplacer des données dans une autre Sous-région ou Région.

Informations générales

Un snapshot créé à un moment donné correspond à une image d’un volume qui contient toutes les informations nécessaires pour créer une réplique de ce volume, de la même taille ou de taille supérieure. Un snapshot peut être utilisé pour sauvegarder vos données ou les partager avec d’autres comptes. 3DS OUTSCALE garantit la persistance de vos snapshots et des données contenues dans ceux-ci.

Tous les snapshots contiennent les informations requises pour créer un nouveau volume avec les données stockées sur le volume d’origine au moment où le snapshot a été créé. Ainsi, si vous supprimez des snapshots précédemment créés, vous pouvez toujours restaurer toutes les données du volume du moment où le snapshot a été créé.

Pour identifier vos ressources plus facilement, vous pouvez leur ajouter des tags. Pour en savoir plus, voir Ajouter ou retirer des tags.

Les snapshots sont placés dans une Région et peuvent être utilisés dans toutes les Sous-régions au sein de cette Région. Vous pouvez copier un snapshot dans une autre Région et ainsi l’utiliser dans les Sous-régions au sein de celle-ci, pour par exemple y créer des copies identiques d’un volume. Pour en savoir plus, voir À propos des Régions et Sous-régions.

Le nombre de snapshots que vous pouvez créer dépend des quotas alloués à votre compte.

Un snapshot peut être dans l’un des états suivants :

  • in-queue : La demande de création du snapshot a été reçue.

  • pending : La création du snapshot est en cours.

  • completed : Le snapshot est prêt et disponible.

    Les données contenues dans un snapshot sont considérées figées lorsque le snapshot est dans l’état completed.

  • error : La création du snapshot a échoué.

  • deleting : La demande de suppression du snapshot a été reçue.

Permissions et copies des snapshots

Vous pouvez partager des snapshots avec d’autres comptes dans la même Région. Ceci leur permet de créer des volumes ou des OUTSCALE machine images (OMI) à partir de vos snapshots, ou de les copier sur leur compte. Vous restez le propriétaire d’un snapshot que vous partagez, dont vous pouvez contrôler l’accès en ajoutant ou retirant des permissions à tout moment. Vous seul pouvez supprimer ce snapshot. Pour en savoir plus, voir Gérer les permissions pour un snapshot.

Vous pouvez également copier un snapshot entre des comptes de la même Région ou de Régions différentes. Contrairement à un snapshot partagé, la copie est indépendante du snapshot source, a son propre ID et appartient au propriétaire du compte destinataire. Vous pouvez ainsi la partager avec d’autres comptes, créer des volumes ou OMI à partir de celle-ci, et la supprimer.

  • Pour copier un snapshot dans la même Région, vous pouvez soit utiliser la méthode export/import décrite ci-dessous, soit partager le snapshot avec un autre compte qui peut ensuite le copier. Pour en savoir plus, voir Copier un snapshot dans la même Région.

  • Pour copier un snapshot entre des Régions différentes, le propriétaire du snapshot source doit d’abord l’exporter sur un bucket OUTSCALE Object Storage (OOS). En utilisant l’URL pré-signée du snapshot, le propriétaire de l’autre compte peut ensuite l’importer, ce qui crée une copie du snapshot sur son compte. Pour en savoir plus, voir Tutoriel : Copier un snapshot dans une Région différente.

Avant d’exporter une OMI ou un snapshot vers une autre Région, assurez-vous que cette action est autorisée par toutes les licences tierces applicables. Si l’export est autorisé, seul l’export d’OMI garantit l’application des licences tierces dans la Région cible, alors que l’export de snapshot ne garantit pas leur application.

  • Vous pouvez exporter un snapshot sur un bucket OOS pour en créer une sauvegarde. Pour en savoir plus, voir Exporter un snapshot sur un bucket OOS.

  • Copier un snapshot dans une autre Région vous permet de créer une sauvegarde de vos données dans différentes Régions isolées et indépendantes les unes des autres.
