Recommandations et limites d’OOS

Cette page fournit des recommandations clés visant à vous aider à maximiser les performances et les avantages de notre solution de stockage d’objets.

Recommandations

Pour une utilisation optimale de vos buckets, nous recommandons :

Usage Recommandation

Buckets

100 par compte (peut être étendu à 1000)

Taille

5 To pour les objets stockables

Versions

10 millions, incluant les delete markers, dans un seul bucket

Téléchargements par seconde, pour les requêtes GET/HEAD

300 pour un seul bucket

Téléchargements par seconde, pour les requêtes PUT/COPY/POST/DELETE

200 pour un seul bucket

Limites absolues

Pour les objets :

Contrainte Description

Durée du téléchargement

Doit effectuer les téléchargements en moins de 3 minutes pour éviter les dépassements de délai.

Les délais de requête sont influencés par le temps nécessaire pour télécharger un fichier : plus le fichier est volumineux et plus la vitesse de téléchargement est lente, plus le risque de dépasser la limite de temps est élevé. Pour télécharger des objets volumineux, nous recommandons de privilégier le dépôt de vos objets en parties. Pour en savoir plus, voir Utiliser un dépôt partitionné.

Tags par objet

Un maximum de 10 tags par objet.

Taille du tag

Un maximum de 128 caractères pour la taille de la clé et 256 caractères pour la taille de la valeur.

ACL par bucket

Seulement une ACL par bucket. Pour en savoir plus, voir Configurer l’ACL d’un objet.
