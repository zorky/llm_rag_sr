À propos des groupes dédiés

Par défaut, les VM sont placées sur des serveurs partagés entre les différents comptes OUTSCALE. Cependant, vous pouvez utiliser des groupes dédiés pour créer vos VM sur des serveurs qui sont alloués uniquement à votre usage.

Les serveurs alloués à un groupe dédié ne sont pas partagés avec des VM associées à d’autres groupes dédiés, avec des VM non dédiées ou avec d’autres comptes.

Cette fonctionnalité est actuellement en bêta. Pour en savoir plus sur les services bêta, voir le Glossaire et nos Conditions générales de ventes et d’utilisation.

Informations générales

Utiliser les groupes dédiés vous permet de bénéficier de la capacité maximum d’un ou plusieurs serveurs, en fonction de la capacité des serveurs et du nombre de VM que vous créez dans vos groupes dédiés. Cela garantit que seules les VM que vous créez dans un groupe dédié sont déployées sur des serveurs qui sont alloués à votre usage.

Vous pouvez créer des groupes dédiés de même configuration. Un serveur spécifique sera assigné à chaque groupe dédié.

Un groupe dédié est composé des éléments suivants :

  • AccountId : l’ID du compte du propriétaire du groupe dédié.

  • CpuGeneration : la génération du processeur des VM du groupe dédié. Pour en savoir plus, voir Types de VM.

  • DedicatedGroupdId : l’ID du groupe dédié. Cet ID doit être utilisé comme allocation des VM ou des Net que vous créez. Pour en savoir plus, voir Créer des VM et Créer un Net.

  • Name : un nom pour le groupe dédié.

  • NetIds : les ID des Nets dans le groupe dédié.

  • SubregionName : la Sous-région dans laquelle le groupe dédié est créé.

  • VmIds : les ID des VM dans le groupe dédié.

Créer des VM dans des groupes dédiés vous permet d’utiliser la virtualisation imbriquée. Pour en savoir plus, À propos des VM et Activer la virtualisation imbriquée.

Groupes dédiés et VM

Vous pouvez créer autant de VM que nécessaire dans un groupe dédié. Si la capacité d’un serveur est dépassée, plusieurs serveurs vous sont alloués pour répondre au nombre de VM demandé.

Pour créer des VM dans un groupe dédié, vous devez spécifier :

  • l’ID d’un groupé dédié comme allocation de la VM.

    Il n’est pas possible de modifier l’allocation d’une VM après sa création. Pour en savoir plus, voir Créer des VM.

  • l’indicateur de performance highest (1). Seules les VM avec cet indicateur de performance sont supportées dans les groupes dédiés. Pour en savoir plus, voir Types de VM.

Si vous souhaitez créer des VM dans un Net et dans un groupe dédié, vous devez créer un Net avec l’ID d’un groupe dédié comme allocation. Ensuite, créez une VM avec le même ID de groupe dédié comme allocation. Sinon, la VM ne peut pas être créée.

Si deux VM sont dans un groupe dédié, mais sont dans deux Net différents, les Nets seront situés sur le même serveur.

Allocation et libération des serveurs

Allocation des serveurs

Un serveur est alloué dès qu’une VM est créée dans un groupe dédié.

Lorsqu’une VM est créée, elle est à l’état pending et les ressources matérielles correspondantes lui sont allouées. Cela inclut l’allocation d’un serveur dédié à votre usage. Pour en savoir plus sur le cycle de vie des VM, voir À propos du cycle de vie des VM.

Vous pouvez obtenir des informations à propos du serveur sur lequel votre VM est créée en accédant aux metadata à partir de la VM elle-même. Pour en savoir plus, voir Accéder aux metadata et user data d’une VM.

Libération des serveurs

Un serveur est libéré lorsque toutes les VMs sur ce serveur sont terminées.

Si toutes les VM d’un serveur alloué à un groupe dédié sont terminées, le serveur est automatiquement libéré. Si vous créez de nouvelles VM dans ce même groupe dédié, un autre serveur peut être alloué pour les nouvelles VM.
