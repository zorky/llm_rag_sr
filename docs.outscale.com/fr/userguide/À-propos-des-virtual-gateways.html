À propos des virtual gateways

Une virtual gateway (passerelle virtuelle) se situe du côté OUTSCALE d’une connexion VPN, et est attachée à l’un de vos Nets.

Son équivalent de votre côté de la connexion VPN est une client gateway. Pour en savoir plus, voir À propos des connexions VPN et À propos des client gateways.

Informations générales

Une virtual gateway est le point d’entrée et de sortie du côté OUTSCALE d’une connexion VPN. Le trafic entre les machines virtuelles (VM) dans le Net et votre réseau d’entreprise est dirigé à travers elle.

Architecture d’une connexion VPN

Vous pouvez attacher une seule virtual gateway à la fois à chaque Net. Cependant, vous pouvez connecter la même virtual gateway à cinq client gateways simultanément. Ceci vous permet de créer des connexions VPN entre le même Net et jusqu’à cinq réseaux d’entreprise, par exemple pour des locaux ou services différents.

Cycle de vie

Une virtual gateway peut être dans l’un des états suivants :

  • Pending : Le processus de création est en cours.

  • Available : La virtual gateway est créée. Elle est ou peut être attachée à un Net.

  • Deleting : Le processus de suppression est en cours.

  • Deleted : La virtual gateway est supprimée.

    Les ressources supprimées restent visibles pendant 1 heure.

De plus, une virtual gateway dans l’état available peut avoir l’un des états d’attachement suivants :

  • Attaching : L’attachement au Net est en cours.

  • Attached : La virtual gateway est attachée au Net et peut être utilisée pour créer une connexion VPN.

  • Detaching : Le détachement du Net est en cours.

  • Detached : La virtual gateway est détachée du Net.
