Tutoriel : Démarrer dans le Cloud OUTSCALE

Dans ce tutoriel, vous apprenez comment créer et vous connecter à votre première machine virtuelle (VM) dans le Cloud 3DS OUTSCALE.

Une VM est une machine virtuelle dans le Cloud public ou dans un Net, un réseau virtuel isolé dans le Cloud. Pour en savoir plus, voir À propos des VM et À propos des Nets.

Ce tutoriel utilise :

  • Cockpit, une interface web développée par 3DS OUTSCALE qui vous permet de créer et gérer vos ressources dans le Cloud 3DS OUTSCALE. Pour en savoir plus, voir À propos de Cockpit.

  • OSC CLI, une interface en ligne de commande (CLI) développée par 3DS OUTSCALE, compatible avec l’API OUTSCALE ainsi que tous les services de l’API AWS-compliant y compris Internal Call Unit (ICU). Pour en savoir plus, voir Installer et configurer OSC CLI.

  • AWS CLI, une CLI développée par Amazon Web Services, compatible avec tous les service de l’API AWS-compliant à l’exception d’Internal Call Unit (ICU). Pour en savoir plus, voir Installer et configurer AWS CLI.

Notre équipe Support est disponible pour vous aider, si nécessaire. Pour en savoir plus, voir Support technique.

Démarrer dans le Cloud 3DS OUTSCALE avec Cockpit v2

  1. (optionnel) Pour vous connecter de manière sécurisée à vos VM, créez une keypair. Pour en savoir plus, voir Créer une keypair.

  2. (optionnel) Pour gérer les flux entrants et sortants de vos VM, créez un security group. Pour en savoir plus, voir Créer un security group.

  3. Créer votre première VM. Pour en savoir plus, voir Créer des VM avec Cockpit v2.
    Votre VM est créée et configurée.

  4. Connectez-vous à votre VM. Pour en savoir plus, voir Accéder à vos VM.

Démarrer dans le Cloud 3DS OUTSCALE avec OSC CLI

Installer et configurer OSC CLI

  • Installer et configurer OSC CLI pour votre compte. Pour en savoir plus, voir Installer et configurer OSC CLI.

Créer votre première keypair

  • Pour vous connecter de manière sécurisée à vos VM, vous devez créer une keypair en utilisant la commande CreateKeypair suivante :

    Exemple de requête
    $ osc-cli api CreateKeypair \
    --KeypairName MyKeypair

    Pour en savoir plus, voir CreateKeyPair dans la documentation de l’API OUTSCALE.

Créer votre premier security group

  1. Pour gérer les flux entrants et sortants de vos VM, vous devez créer un security group en utilisant la commande CreateSecurityGroup suivante :

    Exemple de requête
    $ osc-cli api CreateSecurityGroup \
    --SecurityGroupName my-security-group \
    --Description "SECURITY_GROUP_DESCRIPTION"

    Pour en savoir plus, voir CreateSecurityGroup dans la documentation de l’API OUTSCALE.

  2. Pour autoriser les flux dans un protocole et sur des ports spécifiques, vous devez ajouter des règles au security group en utilisant la commande CreateSecurityGroupRule suivante :

    Exemple de requête
    $ osc-cli api CreateSecurityGroupRule \
    --SecurityGroupId sg-xxxxxxxx \
    --Flow Inbound \
    --IpProtocol tcp \
    --FromPortRange 22 \
    --ToPortRange 22 \
    --IpRange xx.xx.xx.xx/32

    où xx.xx.xx.xx est l’IP de votre ordinateur actuel.

    Pour en savoir plus, voir CreateSecurityGroupRule dans la documentation de l’API OUTSCALE.

Créer et accéder à votre première VM

  1. Pour créer votre première VM, utilisez la commande CreateVms en suivant cette syntaxe :

    Exemple de requête
    $ osc-cli api CreateVms \
    --ImageId ami-xxxxxxxx \
    --KeypairName MyKeypair \
    --SecurityGroupIds '["sg-xxxxxxxx"]' \
    --MinVmsCount 1 \
    --MaxVmsCount 1

    où ami-xxxxxxxx est l’ID de l’OMI à utiliser pour créer la VM. Pour en savoir plus, voir À propos des OMI et Référence des OMI officielles.

    Pour en savoir plus, voir CreateVms dans la documentation de l’API OUTSCALE.

  2. Pour vous connecter à votre VM, voir Accéder à vos VM.

Démarrer dans le Cloud 3DS OUTSCALE avec AWS CLI

Installer et configurer AWS CLI

  • Installez et configurez AWS CLI. Pour en savoir plus, voir Installer et configurer AWS CLI.

Créer votre première keypair

  • Pour vous connecter de manière sécurisée à vos instances, vous devez créer une keypair. Pour cela, utilisez la commande create-key-pair en suivant cette syntaxe :

    Exemple de requête
    $ aws ec2 create-key-pair \
    --profile YOUR_PROFILE \
    --key-name MyKeypair \
    --endpoint https://fcu.eu-west-2.outscale.com

    Pour en savoir plus, voir Créer une keypair.

Créer votre premier security group

  1. Pour gérer les flux entrants et sortants de votre instance, vous devez créer un security group en utilisant la commande create-security-group suivante :

    Exemple de requête
    $ aws ec2 create-security-group \
    --profile YOUR_PROFILE \
    --group-name my-security-group \
    --description SECURITY_GROUP_DESCRIPTION \
    --endpoint https://fcu.eu-west-2.outscale.com

    Pour en savoir plus, voir Créer un security group.

  2. Pour autoriser les flux dans un protocole et sur des ports spécifiques, vous devez ajouter des règles au security group en utilisant la commande authorize-security-group-ingress suivante :

    Exemple de requête
    $ aws ec2 authorize-security-group-ingress \
    --profile YOUR_PROFILE \
    --group-name my-security-group \
    --protocol tcp \
    --port 22 \
    --cidr xx.xx.xx.xx/32 \
    --endpoint https://fcu.eu-west-2.outscale.com

    où xx.xx.xx.xx est l’IP de votre ordinateur actuel.

    Pour en savoir plus, voir Ajouter des règles à un security group.

Lancer et accéder à votre première instance

  1. Pour lancer votre première instance, utilisez la commande run-instances en suivant cette syntaxe :

    Exemple de requête
    $ aws ec2 run-instances \
    --profile YOUR_PROFILE \
    --image-id ami-xxxxxxxx \
    --key-name MyKeypair \
    --security-groups my-security-group \
    --endpoint https://fcu.eu-west-2.outscale.com

    où ami-xxxxxxxx est l’ID de l’OMI à utiliser pour lancer l’instance. Pour en savoir plus, voir À propos des OMI et Référence des OMI officielles.

    Pour en savoir plus, voir Créer des VM.

  2. Pour vous connecter à votre instance, voir Accéder à vos VM.
