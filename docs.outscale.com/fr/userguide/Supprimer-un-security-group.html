Supprimer un security group

Vous pouvez supprimer un security group que vous avez créé si celui-ci n’est plus utilisé pour aucune machine virtuelle (VM) ou dans aucune règle de security group, et s’il ne contient aucune règle.

Vous ne pouvez pas supprimer les security groups par défaut.

Supprimer un security group avec Cockpit v2

Avant de commencer : Assurez-vous que le security group que vous souhaitez supprimer :

  • N’est associé à aucune VM. Pour en savoir plus, voir Obtenir des informations sur vos VM et Terminer des VM.

  • N’est référencé comme security group source ou destination dans aucune règle de security group.

  • Ne contient aucune règle.

Pour en savoir plus, voir Obtenir des informations sur vos security groups et Retirer des règles d’un security group.

  1. Dans le dashboard Security Groups, cochez la case du security group que vous voulez supprimer.

    La multisélection est disponible.

    Le security group est sélectionné.

  2. Cliquez sur  Supprimer.
    Une boîte de dialogue de confirmation apparaît.

  3. Cliquez sur Supprimer.
    Le security group est supprimé.

À ce jour, cette section est disponible en anglais uniquement.

The DeleteSecurityGroup command deletes a specified security group.
You can specify either the name of the security group or its ID.
This action fails if the specified group is associated with a virtual machine (VM) or referenced by another security group.

Request sample
$ osc-cli api DeleteSecurityGroup --profile "default" \
    --SecurityGroupId "sg-12345678"

This command contains the following attributes that you need to specify:

  • DryRun: (optional) If true, checks whether you have the required permissions to perform the action.

  • SecurityGroupId: (optional) The ID of the security group you want to delete.

  • SecurityGroupName: (optional) The name of the security group.

The DeleteSecurityGroup command returns the following elements:

  • ResponseContext: Information about the context of the response.

    • RequestId: The ID of the request.

Result sample
{
  "ResponseContext": {
    "RequestId": "0475ca1e-d0c5-441d-712a-da55a4175157"
  }
}

Supprimer un security group avec AWS CLI

Avant de commencer :

  • Installez et configurez AWS CLI. Pour en savoir plus, voir Installer et configurer AWS CLI.

  • Assurez-vous que le security group que vous souhaitez supprimer :

    • N’est associé à aucune instance. Pour en savoir plus, voir Obtenir des informations sur vos VM et Terminer des VM.

    • N’est référencé comme security group source ou destination dans aucune règle de security group.

    • Ne contient aucune règle.

Pour en savoir plus, voir Obtenir des informations sur vos security groups et Retirer des règles d’un security group.

Pour supprimer un security group, utilisez la commande delete-security-group en suivant cette syntaxe :

Exemple de requête
$ aws ec2 delete-security-group \
    --profile YOUR_PROFILE \
    --group-id sg-12345678 \
    --group-name my-security-group \
    --endpoint https://fcu.eu-west-2.outscale.com

Cette commande contient les attributs suivants que vous devez spécifier :

  • (optionnel) profile : Le profil nommé que vous voulez utiliser, créé pendant la configuration d’AWS CLI. Pour en savoir plus, voir Installer et configurer AWS CLI.

  • (optionnel) group-id : L’ID du security group.

  • (optionnel) group-name : Le nom du security group.

    • Dans un Virtual Private Cloud (VPC), vous devez spécifier l’attribut group-id.

    • Dans le Cloud public, vous pouvez spécifier l’attribut group-id ou group-name.

  • endpoint : Le endpoint correspondant à la Région à laquelle vous voulez envoyer la requête. Pour en savoir plus, voir Installer et configurer AWS CLI.

Le security group spécifié est supprimé.
