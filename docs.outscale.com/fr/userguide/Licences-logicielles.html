Licences logicielles

Il y a plusieurs façons d’utiliser des logiciels sur les machines virtuelles (VM) fournies par 3DS OUTSCALE. Vous devez tout d’abord vous assurer que vous avez le droit d’utiliser le logiciel dont vous avez besoin dans un environnement Cloud tel que celui fourni par 3DS OUTSCALE.

Si vous avez un doute quant à cette question, contactez le fournisseur du logiciel, chacun ayant ses propres règles.

Systèmes d’exploitation

3DS OUTSCALE supporte plusieurs systèmes d’exploitation, ceux-ci devant être compatibles avec des architectures x86 ou x64. 3DS OUTSCALE ne supporte pas encore les architectures ARM. Si vous êtes intéressé par ce type d’architecture, contactez notre service commercial à l’adresse suivante : sales@outscale.com.

La problématique liée aux systèmes d’exploitation est d’avoir de bonnes performances en termes d’inputs et outputs (I/Os), comprenant notamment les I/Os du réseau et des volumes. Linux et Windows permettent ceci à l’aide des drivers VirtIO. Si vous souhaitez installer votre propre système d’exploitation, n’oubliez pas d’utiliser les drivers VirtIO si ceux-ci sont disponibles. Sans ça, les performances de votre VM peuvent être réduites de manière significative. Pour en savoir plus sur comment importer votre propre environnement, voir Migrer une VM Linux vers le Cloud OUTSCALE.

Systèmes d’exploitation Linux

3DS OUTSCALE fournit plusieurs distributions de Linux. Pour en savoir plus à propos de la liste et consulter les ID des OMI officielles disponibles dans chaque Région OUTSCALE, voir Référence des OMI officielles. Pour en savoir plus sur les Régions OUTSCALE disponibles, voir À propos des Régions et Sous-régions.

Il n’y a pas de frais supplémentaires pour l’utilisation des OMI Ubuntu ou CentOS fournies. Des frais supplémentaires sont appliqués pour les OMI Oracle Linux et Redhat Enterprise Linux (RHEL). Pour en savoir plus sur les frais supplémentaires, voir le site officiel de 3DS OUTSCALE.

Systèmes d’exploitation Microsoft Windows

3DS OUTSCALE est un fournisseur Windows sous contrat de licence SPLA, et fournit une OMI officielle Windows Server 2019.

Pour en savoir plus sur la facturation de Windows Server, voir le site officiel de 3DS OUTSCALE.
Pour en savoir plus à propos de la liste et consulter les ID des OMI officielles disponibles dans chaque Région OUTSCALE, voir Référence des OMI officielles.

Middleware

Middleware Microsoft

Les programmes de middleware Microsoft présents dans la liste suivante sont autorisés par le programme SPLA. 3DS OUTSCALE peut fournir la licence, facturée au mois, pour chacun de ces programmes :

  • BizTalk Server*

  • Dynamics AX Server*

  • Dynamics 365*

  • Exchange Server*

  • Forefront Identity Manager*

  • Microsoft Office

  • Project Server*

  • Remote Desktop Services*

  • SharePoint Server*

  • Skype for Business Server*

  • SQL Server Standard Edition*

  • SQL Server Enterprise Edition*

  • SQL Server Business Intelligence Edition*

  • System Center Server*

  • Visio

  • Visual Studio Team Foundation Server*

La licence dépend du type de logiciel. Elle peut être par utilisateur, par cœur, par processeur physique, ou une combinaison de ces éléments. Elle peut nécessiter la souscription aux VM dédiées et non aux VM classiques. Contactez notre département commercial à l’adresse sales@outscale.com si vous avez des questions concernant les tarifs ou les conditions d’utilisation de ces logiciels.

Les licences marquées avec "*" sont compatibles avec le Microsoft License Mobility program. Si vous êtes déjà un client Microsoft et que vous souscrivez à Software Assurance, vous devriez considérer cette option. La principale différence est qu’en cas de problème sur une licence utilisant l’option License Mobility, vous devez contacter le support Microsoft et non le support 3DS OUTSCALE.

Middleware Oracle

3DS OUTSCALE n’est pas un fournisseur autorisé de licences Oracle. Pour utiliser du middleware Oracle dans le Cloud OUTSCALE, vous devez contacter Oracle et acheter votre licence auprès d’eux directement. Plus spécifiquement, vous devez disposer d’un contrat de licence Oracle qui permet l’utilisation de leur solution dans un Cloud public comme celui d’OUTSCALE, par exemple un contrat de type ULA (Unlimited Licence Agreement).

Il est également possible, principalement pour les ISV, d’obtenir une "exception SaaS" qui vous autorise à utiliser Oracle en mode BYOL dans un environnement Cloud. Ceci est réservé spécifiquement aux fournisseurs de SaaS qui utilisent Oracle comme backend et ont besoin d’une autorisation spécifique d’Oracle.

Le Cloud OUTSCALE respecte les règles Oracle en termes d’allocation de ressources.

Middleware d’autres ISV

Chaque ISV a sa propre politique de licence qui inclut ou non le Cloud. Vous pouvez contacter notre service commercial à l’adresse sales@outscale.com pour obtenir de l’aide. Vous pouvez également contacter directement l’ISV, ceux-ci pouvant déjà avoir une licence compatible avec le Cloud disponible.

Logiciels open source

Tout logiciel open source peut être utilisé dans le Cloud selon les termes de sa licence.

Soyez prudent avec les licences AGPL qui peuvent mettre en danger votre propriété intellectuelle, ce type de licence étant viral et se propageant donc aux logiciels interconnectés avec celui que vous utilisez. Beaucoup de licences open source commerciales proposent à la fois des licences AGPL et des licences commerciales non virales. Utilisez celle appropriée pour éviter tout problème.
