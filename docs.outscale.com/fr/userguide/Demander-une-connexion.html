Demander une connexion

Vous pouvez demander une connexion DirectLink pour connecter votre réseau à un site DirectLink que vous spécifiez, où vous devez avoir un routeur ou une fibre optique relié à votre réseau. Cette connection vous permet ensuite d’accéder à vos ressources dans l’ensemble de la Région associée à ce site. Si vos ressources sont placées dans une seule Sous-région, nous vous recommendons de créer une connexion entre votre réseau et le site correspondant pour réduire le phénomène de latence.

Une fois la connexion demandée, 3DS OUTSCALE vous envoie une lettre d’autorisation (Letter of Authorization - LOA) par e-mail vous autorisant à vous connecter au routeur DirectLink correspondant. Vous devez ensuite configurer la connexion avec une ou plusieurs DirectLink interfaces pour accéder à vos Nets à l’aide de la connexion DirectLink.

Demander une connexion avec Cockpit v2

  1. Dans le dashboard DirectLink, cliquez sur  Créer une connexion DirectLink.
    La boîte de dialogue CRÉER UNE CONNEXION DIRECTLINK apparaît.

  2. Dans le champ Nom, tapez le nom de la connexion.

  3. Dans la liste Site, sélectionnez le site requis pour la connexion. Pour en savoir plus, voir Obtenir des informations sur les sites DirectLink.

  4. Dans la liste Bande passante, sélectionnez la bande passante pour la connexion.

  5. Cliquez sur Créer.
    La demande de connexion est envoyée à 3DS OUTSCALE.

Demander une connexion avec OSC CLI

À ce jour, cette section est disponible en anglais uniquement.

The CreateDirectLink command creates a DirectLink between a customer network and a specified DirectLink location.

For more information, see About DirectLink.

Request sample
$ osc-cli api CreateDirectLink --profile "default" \
    --Location "PAR1" \
    --Bandwidth "1Gbps" \
    --DirectLinkName "Connection to Outscale"

This command contains the following attributes that you need to specify:

  • Bandwidth: The bandwidth of the DirectLink (1Gbps | 10Gbps).

  • DirectLinkName: The name of the DirectLink.

  • DryRun: (optional) If true, checks whether you have the required permissions to perform the action.

  • Location: The code of the requested location for the DirectLink, returned by the ReadLocations method.

The CreateDirectLink command returns the following elements:

  • DirectLink: Information about the DirectLink.

    • AccountId: The account ID of the owner of the DirectLink.

    • Bandwidth: The physical link bandwidth (either 1 Gbps or 10 Gbps).

    • DirectLinkId: The ID of the DirectLink (for example, dxcon-xxxxxxxx).

    • DirectLinkName: The name of the DirectLink.

    • Location: The datacenter where the DirectLink is located.

    • RegionName: The Region in which the DirectLink has been created.

    • State: The state of the DirectLink.

  • requested: The DirectLink is requested but the request has not been validated yet.

  • pending: The DirectLink request has been validated. It remains in the pending state until you establish the physical link.

  • available: The physical link is established and the connection is ready to use.

  • deleting: The deletion process is in progress.

  • deleted: The DirectLink is deleted.

  • ResponseContext: Information about the context of the response.

    • RequestId: The ID of the request.

Result sample
{
  "ResponseContext": {
    "RequestId": "0475ca1e-d0c5-441d-712a-da55a4175157"
  },
  "DirectLink": {
    "AccountId": "123456789012",
    "Bandwidth": "1Gbps",
    "DirectLinkId": "dxcon-12345678",
    "DirectLinkName": "Connection to Outscale",
    "Location": "PAR1",
    "RegionName": "eu-west-2",
    "State": "requested"
  }
}

Demander une connexion avec AWS CLI

Avant de commencer : Installez et configurez AWS CLI. Pour en savoir plus, voir Installer et configurer AWS CLI.

Pour demander une connexion, utilisez la commande create-connection en suivant cette syntaxe :

Exemple de requête
$ aws directconnect create-connection \
    --profile YOUR_PROFILE  \
    --location PAR1 \
    --bandwidth 1Gbps \
    --connection-name Connection to Outscale \
    --endpoint https://directlink.eu-west-2.outscale.com

Cette commande contient les attributs suivants que vous devez spécifier :

  • (optionnel) profile : Le profil nommé que vous voulez utiliser, créé pendant la configuration d’AWS CLI. Pour en savoir plus, voir Installer et configurer AWS CLI.

  • location : Le code du site requis pour la connexion. Pour en savoir plus, voir Obtenir des informations sur les sites DirectLink.

  • bandwidth : La bande passante pour la connexion (1Gbps ou 10Gbps).

  • connection-name : Le nom de la connexion.

  • endpoint : Le endpoint correspondant à la Région à laquelle vous voulez envoyer la requête. Pour en savoir plus, voir Installer et configurer AWS CLI.

La commande create-connection renvoie les éléments suivants :

  • ownerAccount : L’ID de compte du propriétaire de la connexion.

  • connectionID : L’ID de la connexion.

  • connectionState : L’état de la connexion. Pour en savoir plus, voir À propos de DirectLink.

  • bandwidth : La bande passante pour la connexion.

  • location : Le site où se trouve la connexion.

  • connectionName : Le nom de la connexion.

  • region : La Région où se trouve la connexion.

Exemple de résultat
{
    "ownerAccount": "123456789000",
    "connectionId": "dxcon-12345678",
    "connectionState": "requested",
    "bandwidth": "1Gbps",
    "location": "PAR1",
    "connectionName": "Connection to Outscale",
    "region": "eu-west-2"
}

La demande de connexion est envoyée à 3DS OUTSCALE.
