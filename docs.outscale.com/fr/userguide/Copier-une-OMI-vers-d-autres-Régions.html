Copier une OMI vers d’autres Régions

Vous pouvez copier une image machine OUTSCALE (OMI) entre plusieurs comptes situés dans des Régions différentes avec une méthode d’export/import.

Le propriétaire de l’OMI source doit d’abord exporter l’OMI vers un bucket OUTSCALE Object Storage (OOS). L’autre compte doit ensuite l’importer à l’aide de l’URL pré-signée du fichier manifeste de l’OMI.

Avant d’exporter une OMI ou un snapshot vers une autre Région, assurez-vous que cette action est autorisée par toutes les licences tierces applicables. Si l’export est autorisé, seul l’export d’OMI garantit l’application des licences tierces dans la Région cible, alors que l’export de snapshot ne garantit pas leur application.

  1. Le propriétaire de l’OMI source doit :

    1. Exporter l’OMI vers un bucket OOS. Pour en savoir plus, voir Exporter une OMI vers un bucket.
      Cette action crée dans le bucket les snapshots nécessaires ainsi qu’un fichier manifeste.

    2. Fournir à l’autre utilisateur une URL pré-signée vers le fichier manifeste. Pour en savoir plus, voir Créer une URL pré-signée.

    Au lieu d’exporter l’OMI en tant que telle, le propriétaire peut aussi manuellement rédiger le fichier manifeste correspondant à l’OMI, puis le déposer manuellement ainsi que toutes les snapshots nécessaires dans le bucket, mais cela requiert davantage d’étapes. Pour en savoir plus, voir Référence des fichiers manifestes.

  2. L’autre utilisateur peut ensuite importer l’OMI depuis le bucket. Pour en savoir plus, voir Importer une OMI depuis un bucket.

    Une copie de l’OMI est créée dans le compte de l’utilisateur.
