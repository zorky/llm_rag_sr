Notes de releases API (ancienne page)

Cette page liste les nouvelles fonctionnalités incluses dans chaque release de TINA OS.

2024

Juin 2024

Depuis le 17 juin 2024, les notes de mise à jour se trouvent sur une nouvelle page : Notes de mise à jour.

Avril 2024

22 avril 2024 (eu-west-2) / 22 avril 2024 (cloudgouv-eu-west-1) / 15 avril 2024 (us-west-1) / 17 avril 2024 (us-east-2) / 16 avril 2024 (ap-northeast-1)

  • Dans l’API OUTSCALE, vous pouvez désormais gérer des groupes pour les utilisateurs EIM avec les méthodes UserGroup. Les groupes vous permettent d’accorder des autorisations à plusieurs utilisateurs, facilitant ainsi la gestion des autorisations. Pour en savoir plus, voir À propos des groupes EIM.

  • Vous pouvez désormais gérer des politiques pour vos groupes d’utilisateurs avec les méthodes Policy suivantes :

    • DeleteUserGroupPolicy

    • LinkManagedPolicyToUserGroup

    • PutUserGroupPolicy

    • ReadManagedPoliciesLinkedToUserGroup

    • ReadUserGroupPolicies

    • ReadUserGroupPolicy

    • UnlinkManagedPolicyFromUserGroup

  • Vous pouvez désormais modifier la description d’une OMI avec le paramètre Description de la méthode API OUTSCALE UpdateImage.

  • Les méthodes API OUTSCALE CreateNatService et ReadNatServices supportent désormais le paramètre ClientToken

  • La méthode API OUTSCALE ReadNics supporte désormais le filtre LinkPublicIpPublicDnsNames.

  • Les réponses de la méthode API OUTSCALE ReadConsumptionAccount sont désormais renvoyées dans la devise de votre compte.

  • Dans l’API Flexible Compute Unit (FCU), les filtres suivants sont retirés de la méthode DescribeInstance :

    • dns-name

    • private-dns-name

    • network-interface.private-dns-name

  • Vous pouvez désormais paginer les réponses des méthodes API OUTSCALE suivantes :

    • ReadClientGateways

    • ReadDedicatedGroups

    • ReadDirectLinkInterfaces

    • ReadDirectLinks

    • ReadImageExportTasks

    • ReadInternetServices

    • ReadKeypairs

    • ReadLocations

    • ReadNetAccessPointServices

    • ReadNets

    • ReadProductTypes

    • ReadPublicIpRanges

    • ReadQuotas

    • ReadSecurityGroups

    • ReadSnapshotExportTasks

    • ReadSnapshots

    • ReadSubnets

    • ReadSubregions

    • ReadTags

    • ReadVmsState

    • ReadVmTypes

Janvier 2024

15 janvier 2024 (eu-west-2) / 16 janvier 2024 (cloudgouv-eu-west-1) / 8 janvier 2024 (us-west-1) / 10 janvier 2024 (us-east-2) / 9 janvier 2024 (ap-northeast-1)

  • Vous pouvez désormais récupérer le coût unitaire (UnitPrice) et le coût total (Price) d’une ressource consommée, grâce au paramètre ShowPrice de la méthode ReadConsumptionAccount.

  • Dans l’API OUTSCALE, vous pouvez désormais créer des groupes dédiés pour créer des VM dans des groupes sur des servers dédiés.
    Cela vous permet de bénéficier de la capacité maximum du server et vous garantit que seules les VM créées dans des groupes dédiées sont placées sur un ou plusieurs des serveurs dédiés. Pour en savoir plus, voir À propos des groupes dédiés.

    Cette fonctionnalité est disponible en beta.

  • Vous pouvez désormais gérer les policies pour les utilisateurs EIM avec les méthodes Policy.

  • Vous pouvez désormais modifier la route table associée à un subnet specifique dans un Net via la méthode UpdateRouteTableLink dans l’API OUTSCALE. Pour en savoir plus, voir Changer la route table attachée à un Subnet.

  • Vous pouvez désormais paginer les réponses des méthodes API OUTSCALE suivantes :

    • ReadDhcpOptions

    • ReadImages

    • ReadNatServices

    • ReadNetAccessPoints

    • ReadNetPeerings

    • ReadPublicIps

    • ReadRouteTables

    • ReadVirtualGateways

    • ReadVms

    • ReadVolumes

    • ReadVpnConnections

  • Les méthode API OUTSCALE suivantes supportent désormais de nouveaux filtres :

    • ReadImages : ProductCodeNames.

    • ReadNetPeerings : ExpirationDates.

    • ReadSubregions : RegionsNames and States.

    • ReadVmTypes : EphemeralTypes, Eths and Gpus.

    • ReadVms : 60 nouveaux filtres sont supportés, pour en savoir plus voir la documentation API OUTSCALE.

  • Seules les keypairs RSA peuvent déchiffrer le mot de passe d’une VM Windows. Pour en savoir plus, voir Accéder à une VM Windows.

Avril 2023

11 avril 2023 (eu-west-2) / 11 avril 2023 (cloudgouv-eu-west-1) / 3 avril 2023 (us-west-1) / 5 avril 2023 (us-east-2) / 3 avril 2023 (ap-northeast-1)

  • Dans l’API OUTSCALE, la nouvelle méthode ReadCatalogs vous permet d’obtenir la liste des prix des services et ressources OUTSCALE dans une période déterminée (jusqu’à 3 ans). Pour en savoir plus, voir Obtenir le prix de vos ressources.

  • La méthode ReadSnapshots supporte désormais les filtres FromCreationDate et ToCreationDate pour spécifier une période de création.

  • Dans l’API OUTSCALE, le format datetime est désormais renommé date-time.

  • Les méthodes API OUTSCALE suivantes supportent désormais les deux formats date et date-time :

    • CreateAccessKey

    • UpdateAccessKey

    • ReadConsumptionAccount

    • ReadApiLogs

Février 2023

27 février 2023 (eu-west-2) / 27 février 2023 (cloudgouv-eu-west-1) / 20 février 2023 (us-west-1) / 22 février 2023 (us-east-2) / 20 février 2023 (ap-northeast-1)

  • Dans l’API DirectLink, l’Autonomous System Number (ASN) des virtual interfaces privées doit désormais être compris entre 64512 et 65534. Les connexions existantes ne sont pas concernées.

  • Dans l’API OUTSCALE, la méthode UpdateDirectlinkInterface supporte désormais uniquement la valeur 1500 pour le paramètre Mtu.

  • Les comptes de la Région cloudgouv-eu-west-1 sont maintenant avertis par e-mail 13 jours, 8 jours et 1 jour avant l’expiration de leur mot de passe. Pour en savoir plus, voir À propos de la protection par mot de passe.

  • De nouvelles méthodes pour utiliser une même connexion DirectLink avec différents comptes sont désormais disponibles. La méthode AllocatePrivateVirtualInterface vous permet de partager votre connexion DirectLink avec un autre compte OUTSCALE via la création d’une virtual interface privée pour leur compte. Inversement, la méthode ConfirmPrivateVirtualInterface vous permet d’utiliser la connexion DirectLink d’un autre compte OUTSCALE qui a créé une virtual interface privée pour votre compte. Pour en savoir plus, voir AllocatePrivateVirtualInterface et ConfirmPrivateVirtualInterface.

Février 2018

14 février 2018 (eu-west-2) / 13 février 2018 (us-west-1) / 12 février 2018 (us-east-2) / 12 février 2018 (cn-southeast-1)

  • La méthode ReadPublicIpRanges vous permet de lister les adresses IPv4 publiques d’une Région. Pour en savoir plus, voir ReadPublicIpRanges.

  • Les méthodes ReadCatalog et ReadPublicCatalog renvoient désormais les prix pour la Région de votre compte uniquement. L’attribut Region pour spécifier une autre Région n’est plus disponible. Pour en savoir plus, voir ReadCatalog ou ReadPublicCatalog.

  • Par défaut, le trafic entre deux security groups est maintenant autorisé à la fois à travers les IP publiques et privées. Pour en savoir plus, voir AuthorizeSecurityGroupEgress et AuthorizeSecurityGroupIngress.

    Pour restreindre le trafic aux IP privées uniquement, contactez notre équipe Support : support@outscale.com.

Décembre 2017

6 décembre 2017 (eu-west-2) / 5 décembre 2017 (us-west-1) / 4 décembre 2017 (us-east-2) / 5 décembre 2017 (cn-southeast-1)

  • Les listener rules vous permettent de rediriger le trafic entre un load balancer et ses VM backend à partir du chemin d’accès de la requête. Pour en savoir plus, voir les six nouvelles méthodes suivantes :

    • CreateListenerRule

    • DeleteListenerRule

    • ReadListenerRules

    • UpdateListenerRule

    • RegisterInstancesWithListenerRule

    • DeregisterInstancesFromListenerRule.

  • Un nouveau type d’instances GPU Commodity est disponible : les instances gp3. Pour en savoir plus, voir Types de VM et RunInstances.

  • Le nouvel attribut InstanceInitiatedShutDownBehavior vous permet de définir le comportement de votre instance lorsque vous l’arrêtez ou la terminez. Par défaut ou si paramétré sur stop, l’instance est arrêtée. Si paramétré sur restart, l’instance est arrêtée puis automatiquement redémarrée. Si paramétré sur terminate, l’instance est arrêtée puis terminée. Pour en savoir plus, voir RunInstances ou ModifyInstanceAttribute.

  • La méthode CreateNatGateway crée une NAT gateway dans un subnet public d’un VPC. Pour en savoir plus, voir CreateNatGateway.

Juin 2017

23 juin 2017 (eu-west-2) / 27 juin 2017 (us-west-1) / 26 juin 2017 (us-east-2) / 26 juin 2017 (cn-southeast-1)

  • Une nouvelle méthode a été mise en place pour copier les snapshots et les OUTSCALE machine images (OMI) entre différentes Régions. Désormais, vous devez tout d’abord exporter le snapshot ou l’OMI dans un bucket OSU. L’autre compte doit ensuite l’importer dans l’autre Région. Cette fonctionnalité vous permet également de créer des sauvegardes de vos snapshots ou de vos OMI, ou de copier des snapshots ou des OMI au sein de la même Région. Pour en savoir plus, sur la copie inter-Régions pour les snapshots, voir CreateSnapshotExportTask et ImportSnapshot. Pour en savoir plus sur la copie inter-Régions pour les OMI, voir CreateImageExportTask et RegisterImage.
    Pour les copies intra-Région, les méthodes CopySnapshot et CopyImage sont toujours disponibles.

  • Les Network Address Translation (NAT) gateways sont maintenant créées pour un subnet uniquement. Ainsi, vous devez créer une NAT gateway par subnet pour lequel vous souhaitez autoriser la connexion indirecte à Internet. Pour en savoir plus, voir CreateNatGateway.
