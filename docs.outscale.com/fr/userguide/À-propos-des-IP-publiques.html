À propos des IP publiques

Une IP publique est une adresse IPv4 que vous pouvez allouer à votre compte, pour ensuite l’associer à l’une de vos ressources.

Vous pouvez dissocier une IP publique de sa ressource à n’importe quel moment, puis la réassocier à une autre. Vous pouvez également fixer une IP publique à une machine virtuelle (VM) même en cas d’arrêt et de redémarrage de celle-ci à l’aide d’un tag 3DS OUTSCALE. Pour en savoir plus, voir Configurer une VM avec les user data et les tags OUTSCALE.

Informations générales

Les IP publiques sont des adresses IPv4 publiques que vous pouvez allouer à votre compte. Vous pouvez ensuite les associer aux ressources suivantes :

  • VM

  • Interfaces réseau

  • Load balancers reliés à Internet

  • NAT services

Les IP publiques permettent de communiquer avec Internet. Pour en savoir plus, voir À propos des Nets > IP et accès à Internet.

Pour associer une IP publique dans un Net, celui-ci doit avoir un internet service attaché. Pour en savoir plus, voir Connecter des VM dans un Net à Internet.

Vous pouvez dissocier et réassocier une IP publique à n’importe quel moment. Par exemple, en cas de panne d’une VM, vous pouvez ainsi transférer l’IP publique vers une autre VM. La nouvelle VM a donc la même IP publique, ce qui signifie que les scripts et les applications qui utilisent cette IP publique ne rencontrent aucun problème. Pour en savoir plus, voir la section Association des IP publiques ci-dessous.

Une IP publique dissociée reste allouée à votre compte. Si vous n’utilisez plus une IP publique, vous pouvez la libérer afin de la rendre disponible pour d’autres comptes. Pour en savoir plus, voir Détacher une IP publique d’une VM ou d’une interface réseau et Libérer une IP publique de votre compte.

Assurez-vous qu’aucune application ou qu’aucun script n’utilise l’IP publique que vous voulez libérer.

Association des IP publiques

Vous ne pouvez associer une IP publique qu’à une seule ressource à la fois.

Si vous associez une IP publique à une VM ou à une interface réseau, avec Cockpit vous devez toujours dissocier l’IP publique avant de pouvoir la réassocier à une autre ressource. Avec les API, vous pouvez configurer un comportement de réassociation : soit empêcher celle-ci, soit dissocier automatiquement l’IP publique et la réassocier à la nouvelle VM ou interface réseau que vous spécifiez. Pour en savoir plus, voir Attacher une IP publique à une VM ou à une interface réseau.

Si vous créez un NAT service, vous devez spécifier une IP publique. Pour modifier son IP publique, vous devez supprimer le NAT service puis le recréer avec la nouvelle IP publique. Pour en savoir plus, voir Créer un NAT service.

Vous pouvez aussi associer une IP publique avec un load balancer relié à Internet lorsque vous gérez le load balancer. Pour en savoir plus sur le comportement d’association, voir À propos des load balancers > IP publique. Pour en savoir plus sur la gestion du load balancer, voir Créer un load balancer relié à Internet dans le Cloud public ou Créer un load balancer relié à Internet dans un Net.

Dans le Cloud public

L’IP publique associée remplace l’IP publique par défaut de la VM. Vous pouvez associer une IP publique à une VM avec l’une des solutions suivantes :

  • Association temporaire :
    Cette solution associe instantanément l’IP publique à une VM en marche. Cependant, l’IP publique est dissociée de la VM à chaque fois que vous arrêtez et redémarrez la VM. Pour en savoir plus, voir Attacher une IP publique à une VM ou à une interface réseau.

  • Association persistante :
    Cette solution implique d’utiliser un tag, osc.fcu.eip.auto-attach. Ce tag vous permet d’associer une IP publique à une VM et d’y fixer l’IP publique même en cas d’arrêt et de redémarrage. Ce tag n’est pris en compte qu’au boot, c’est-à-dire à la création ou après un arrêt et redémarrage de la VM. Vous pouvez ajouter ce tag en utilisant les tags classiques ou dans les user data de la VM (avec osc.fcu.eip.auto-attach comme clé et l’IP publique comme valeur). Pour en savoir plus, voir Ajouter des tags à vos ressources ou Configurer une VM avec les user data et les tags OUTSCALE.

    • Si vous n’avez pas le temps d’arrêter et de démarrer votre VM pour y associer une IP publique en utilisant le tag osc.fcu.eip.auto-attach, vous pouvez associer l’IP publique en utilisant la solution d’association temporaire, et ensuite ajouter ce tag à la VM en utilisant la même IP publique comme valeur. Ainsi, l’IP publique est fixée à la VM la prochaine fois que vous arrêtez ou démarrez la VM.

    • Si vous ajoutez le tag osc.fcu.eip.auto-attach à une VM qui a déjà une IP publique associée ou si vous remplacez la valeur du tag avec une autre IP publique, l’IP publique actuelle sera dissociée et remplacée par la nouvelle IP publique que vous spécifiez comme valeur au prochain arrêt et démarrage.

    • Comme les interfaces réseau ne suivent pas le cycle de l’arrêt et démarrage, l’association entre une IP publique et une interface réseau est toujours persistante.

Dans un Net

L’IP publique s’attache à la VM ou à l’interface réseau principale. Si vous travaillez avec plusieurs interfaces réseau, vous pouvez choisir d’associer l’IP publique à n’importe laquelle des network interface cards (NIC). Pour en savoir plus, voir À propos des NIC.
Dans un Net, les associations des IP publiques sont toujours persistantes. Vous pouvez associer une IP publique à une VM avec l’une des solutions suivantes :

  • En utilisant un tag :
    Cette solution implique d’utiliser un tag, osc.fcu.eip.auto-attach. Ce tag vous permet d’associer une IP publique à une VM et d’y fixer l’IP publique même en cas d’arrêt et de redémarrage. Ce tag n’est pris en compte qu’au boot, c’est-à-dire à la création ou après un arrêt et redémarrage de la VM. Vous pouvez ajouter ce tag en utilisant les tags classiques ou dans les user data de la VM (avec osc.fcu.eip.auto-attach comme clé et l’IP publique comme valeur). Pour en savoir plus, voir Ajouter des tags à vos ressources ou Configurer une VM avec les user data et les tags OUTSCALE.

    • Si vous ajoutez le tag osc.fcu.eip.auto-attach à une VM qui a déjà une IP publique associée ou si vous remplacez la valeur du tag avec une autre IP publique, l’IP publique actuelle sera dissociée et remplacée par la nouvelle IP publique que vous spécifiez comme valeur au prochain arrêt et démarrage.

  • En utilisant la méthode UpdateSubnet :
    Cette solution implique de passer le paramètre MapPublicIpOnLaunch du Subnet sur True. Ce paramètre attribue une IP aléatoire à la VM dans le Net et la fixe même lors du processus d’arrêt et redémarrage de la VM. Pour en savoir plus, voir la méthode UpdateSubnet.
